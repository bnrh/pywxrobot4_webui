# 快速开始

## 1. 项目组成

wxrobot_webui 不是 wxrobot_api 本体，而是运行在其旁边的一层 Web UI 与插件服务。

- wxrobot_api：负责微信能力和底层接口。
- wxrobot_webui：负责接收回调、管理插件、提供控制台、持久化配置，并把 AI 模型接到 wxrobot_api 的工具面上。

## 2. 运行前提

建议准备一个独立的 Python 环境，并确保以下依赖可用：

- FastAPI
- uvicorn
- loguru
- pydantic

如果当前环境还没有这些依赖，可以按需安装：

```powershell
pip install fastapi uvicorn loguru pydantic
```

如果你同时会使用 AI 相关能力，还需要保证对应模型厂商的 API Key 已准备好。

## 3. 默认端口与关键配置

项目默认值来自 [config.py](../config.py)：

| 配置项 | 默认值 | 说明 |
| --- | --- | --- |
| Web UI 监听地址 | `127.0.0.1` | 控制台和回调服务监听地址 |
| Web UI 监听端口 | `28080` | 控制台访问端口 |
| 回调路径 | `/messages` | wxrobot_api 推送消息的入口 |
| wxrobot_api 地址 | `http://127.0.0.1:23235` | 插件与 AI 工具调用的目标服务 |
| Worker 数量 | `2` | 后台消息处理协程数 |
| 队列长度 | `1000` | 消息回调入队上限 |
| 心跳间隔 | `30` 秒 | 登录微信账号巡检频率，`0` 表示关闭 |

## 4. 启动顺序

### 步骤一：启动 wxrobot_api

先按 wxrobot_api 自身的方式启动主服务，确保它已经能正常响应接口。

### 步骤二：配置消息回调地址

在 `config.ini` 的 `[base]` 段中，将 `post_api` 指向本项目的回调地址：

```ini
[base]
post_api = http://127.0.0.1:28080/messages
```

如果你改了监听地址、端口或回调路径，需要同步修改这里。

### 步骤三：启动 wxrobot_webui

本仓库当前的真实启动入口是 [main.py](../main.py)：

```powershell
python main.py
```

启动后，服务会打印当前建议写入 `post_api` 的完整地址。

### 步骤四：打开控制台

默认访问地址：

```text
http://127.0.0.1:28080/
```

## 5. 首次接入建议

首次启动后，建议按下面顺序检查：

1. 打开“仪表盘”确认服务已启动、队列为空、已加载插件数正常。
2. 打开“用户管理”确认心跳能拿到当前已登录微信账号。
3. 打开“消息插件”确认默认插件状态与配置是否符合预期。
4. 在 wxrobot_api 侧发送一条测试消息，检查“消息中心”是否能看到消息及插件结果。

## 6. 配置与数据存储

项目运行时主要使用以下目录和文件：

- [webui.sqlite3](../webui.sqlite3)：系统设置、插件启停、插件配置、插件状态、AI 对话与相关数据。连接复用、写锁与批量 commit 说明见 [架构与性能优化](optimizations.md#3-数据库与并发)。
- `logs/`：系统日志文件目录，由运行时生成。
- `uploads/`：结构化配置表单上传的插件素材目录，由运行时生成。

当前存储模型有两个特点：

1. 系统设置和插件配置优先从 SQLite 读取。
2. 第一次运行时会从旧版 `config.ini` 的 `[webui]` 段迁移数据到 SQLite。

若修改了 `frontend/` 或 `static/js` 源码，发布前执行 `npm run build`（Vite + 资源版本戳）。未改前端时可直接启动后端。

## 7. 健康检查与常用入口

- `/`：前端控制台首页。
- `/health`：基础健康检查。
- `/api/overview`：运行概览数据。
- `/api/plugins`：插件列表与配置元数据。
- `/api/settings`：系统设置读写入口。

## 8. 常见问题

### 启动时报缺少 `uvicorn`

说明当前 Python 环境缺少 Web 服务依赖。先安装依赖，再重新执行 `python main.py`。

### 页面能打开，但一直提示加载失败

优先检查以下几项：

1. wxrobot_api 是否已经启动。
2. `wxrobot_api_base_url` 是否填写正确。
3. `post_api` 是否已经指向本项目回调地址。
4. 浏览器访问的是否就是当前服务监听地址与端口。

### 修改了系统设置但没有立即生效

部分字段支持热重载，部分字段需要重启。控制台“系统设置”页和“仪表盘”会提示哪些字段仍然需要重启服务才能生效。
