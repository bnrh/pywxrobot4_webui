# 控制台说明

前端控制台由 [frontend/index.html](../frontend/index.html) 与 [static/js](../static/js) 模块提供。页面骨架按 Tab / Modal 拆成片段动态加载；首屏只请求概览与当前 Tab 数据。构建产物通过 Vite 生成，并用 git hash 做资源版本戳（见 [架构与性能优化](optimizations.md#4-前端刷新与首屏)）。

当前主要包含 9 个页面。

## 页面总览

| 页面 | 作用 | 常见操作 |
| --- | --- | --- |
| 仪表盘 | 查看运行概览、队列、插件加载数、心跳状态 | 刷新视图、重载运行配置 |
| 消息中心 | 查看最近进入插件服务的消息和插件处理结果 | 检查回调是否到达、排查插件链路 |
| 用户管理 | 查看心跳检测到的登录微信账号 | 确认 wxpid、昵称、账号在线状态 |
| 功能插件 | 执行一次性功能插件或周期性任务插件 | 导出、汇总、批量整理等 |
| 智能插件 | 配置 AI 模型并调用 pywxrobot4 工具 | 发起对话、切换模型、查看工具列表 |
| 消息插件 | 启停消息插件、编辑结构化配置 | 开关插件、保存配置、热重载 |
| 插件日志 | 查看插件输出的结构化日志 | 按插件、级别、关键词筛选 |
| 系统设置 | 修改监听地址、回调路径、API 地址等 | 保存设置、识别是否需要重启 |
| 日志中心 | 查看日志文件最新内容 | 按时间范围、模块、关键词过滤 |

## 1. 仪表盘

“仪表盘”展示的是运行时的全局状态，而不是静态配置。

这里重点看几项：

- 当前回调地址：确认 pywxrobot4 的 `post_api` 应该指向哪里。
- 已启用插件数 / 已加载插件数：排查配置已保存但插件未成功载入的情况。
- 队列长度与 Worker 数：判断是否有消息积压。
- 心跳状态：确认当前是否拿到了已登录微信账号。
- 运行时指标：队列拒绝数、最近消息 / 插件日志规模等。
- 待重启字段：识别哪些系统设置修改后还未完全生效。

消息中心与部分指标在 SSE 正常时会事件驱动刷新，减少与轮询重叠；概览仍保留较长间隔的兜底刷新。

## 2. 消息中心

“消息中心”是排查回调链路最直接的页面。每条消息会保留：

- `msgid`
- `conversation_wxid`
- `sender_wxid`
- `msg_type` / `local_type`
- 是否群聊
- 原始内容摘要
- 插件处理结果列表

适合用来回答三个问题：

1. pywxrobot4 是否真的把消息推送过来了。
2. 哪个插件处理了这条消息。
3. 插件是跳过、处理成功还是处理失败。

## 3. 用户管理

“用户管理”基于服务端心跳检查结果展示当前登录账号。它依赖系统设置里的 `heartbeat_interval_seconds`。

常见用途：

- 确认当前有哪些微信进程在线。
- 为多开场景选择正确的 wxpid。
- 在功能插件导出、多账号切换、群成员读取失败时判断是否是账号在线状态问题。

## 4. 功能插件

“功能插件”面向手动执行类插件，例如导出联系人、导出群成员、消息汇总、群成员去重、批量标签整理等。

这里有两个执行模型：

1. 先打开执行弹窗，再传入本次执行配置。
2. 对声明了 `direct_execute = True` 的插件，点击执行时直接使用已保存配置运行（例如导出联系人、消息汇总、近期图片下载、防撤回）。

`direct_execute` / `message_summary` 来自插件模块元数据，而不是服务端硬编码列表。新增同类插件时在插件文件中声明即可。

执行完成后，页面会刷新概览和插件状态，便于马上确认输出结果。

## 5. 智能插件

“智能插件”是一个面向运营与管理场景的 AI 工具代理页面。

它支持：

- 配置多个 AI 厂商和多个配置项。
- 配置多个可切换的提示词插件，例如群聊总结、群发言统计。
- 为每次对话选择模型。
- 为每次对话选择提示词插件。
- 持久化对话历史。
- 以异步任务方式执行长对话，并支持手动停止。
- 查看模型可调用的工具列表。

当前页面还包含独立的对话管理能力：

- 新建对话
- 切换对话
- 清空对话
- 查看进行中的 AI 任务状态

## 6. 消息插件

“消息插件”页负责管理所有依赖消息触发的插件。

页面内能做的事情包括：

- 启用或停用插件。
- 编辑插件配置。
- 使用结构化表单，而不是手写 JSON。
- 触发运行时热重载。

当前结构化配置表单支持的能力已经比旧版 README 里描述的多得多，常见包括：

- 普通文本、数字、勾选框、下拉框。
- 可搜索下拉框。
- `object-list` 表格式规则配置。
- `multi-checkbox` 多选消息类型。
- `string-list` 文本列表。
- 插件图片上传。
- 群成员选择器等增强型字段。

## 7. 插件日志

这个页面展示的是插件结构化日志，不是原始文件日志。

日志项包含：

- 插件模块
- 插件名
- 级别
- 作用域
- 消息文本
- 结构化数据对象

相比日志文件，这里更适合按插件和关键词快速定位行为。

## 8. 系统设置

“系统设置”面向服务运行参数，而不是单个插件配置。

常见配置包括：

- 监听地址和端口
- 回调路径
- pywxrobot4 地址
- 请求超时
- Worker 数量
- 队列长度
- 心跳间隔

保存后，系统会自动判断哪些字段可以热重载，哪些字段仍需重启。

## 9. 日志中心

“日志中心”直接读取日志文件目录，支持：

- 选择日志文件
- 按最近 1 小时 / 6 小时 / 1 天等范围过滤
- 按日志级别过滤
- 按模块名或关键词过滤

适合在插件日志之外，查看系统级异常和底层运行日志。

## 常见操作流程

### 启用一个新插件

1. 进入“消息插件”或“功能插件”页面。
2. 找到目标插件并启用。
3. 打开配置弹窗，保存结构化配置。
4. 如有需要，点击“重载运行配置”。
5. 到“消息中心”或“插件日志”检查实际效果。

### 排查一条消息为什么没有触发插件

1. 在“消息中心”确认消息是否已经进队列。
2. 看 `msg_type` / `local_type` 和会话类型是否匹配插件规则。
3. 到“消息插件”检查插件是否启用、配置是否命中。
4. 到“插件日志”查看插件是否输出了跳过、告警或错误信息。

### 执行一个导出类插件

1. 进入“功能插件”。
2. 配置 wxpid、时间范围、群聊或好友范围。
3. 执行插件。
4. 根据插件返回结果到导出目录查看 CSV 或文本输出。
